🔍 Test Documents-Simple Vocabulary Save

📋 Hướng dẫn kiểm tra

  1. Mở trang /dashboard-new/documents-simple trong tab khác
  2. Mở Developer Console (nhấn F12 hoặc Ctrl+Shift+I)
  3. Chọn tab Console trong Developer Tools
  4. Upload một file PDF hoặc DOCX
  5. Quan sát các log messages trong console
  6. Copy tất cả log messages và gửi cho developer

⚠️ Các log cần chú ý

Bạn sẽ thấy các log messages sau trong console:

  • [Upload] About to save vocabulary to database - Xác nhận data structure trước khi save
  • [DEBUG handleSaveToDatabase] Function called with data - Xác nhận function được gọi
  • [DEBUG handleSaveToDatabase] Vocabulary to save: X items - Số lượng từ vựng
  • [Save] Processing item X/Y - Từng item đang được xử lý
  • [Save] Payload for "word" - Dữ liệu gửi lên API
  • [Save] Response status for "word" - Kết quả từ API
  • [Save] Final results - Saved: X Failed: Y - Kết quả cuối cùng

📝 Các trường hợp có thể xảy ra

✅ Trường hợp 1: Không có log nào

→ Function không được gọi. Có thể do:

  • Upload thất bại trước khi đến bước save
  • Response từ API không có vocabulary data
⚠️ Trường hợp 2: Có log "Vocabulary to save: 0 items"

→ Data structure không đúng. Kiểm tra:

  • Response từ /api/upload-document-complete
  • Field name: data.vocabulary hoặc data.flashcards
❌ Trường hợp 3: Có log "Response status: 401"

→ Không có session/authentication. Cần:

  • Đăng nhập lại
  • Kiểm tra session trong /api/vocabulary
❌ Trường hợp 4: Có log "Response status: 500"

→ Lỗi server. Kiểm tra:

  • MongoDB connection
  • Server logs trong terminal
✅ Trường hợp 5: Có log "Success response"

→ Save thành công! Kiểm tra:

  • Vào /dashboard-new/vocabulary để xem từ vựng
  • Filter by source: "document"
